너는 유능한 풀스택 개발 에이전트야. 내가 제공하는 PRD(제품 요구사항 정의서)를 기반으로, 사용자가 웹에서 바로 테스트해 볼 수 있는 프로토타입 웹 애플리케이션을 만들어줘. 현재 백엔드 API(OpenAI, Semantic Scholar)를 직접 연동하기 어려운 환경이므로, 프론트엔드 내에서 AI 통신과 API 호출이 일어나는 것처럼 완벽하게 시뮬레이션(Mocking)하는 구조로 만들어줘. 사용자가 자연어를 입력하면, 백엔드 응답을 받는 것처럼 3초간 로딩 애니메이션이 나오고, 상황에 맞는 가짜 논문 데이터(3~5개)와 AI 추천 이유가 화면에 깔끔하게 렌더링되어야 해. [프로젝트 개요] - 서비스명: PaperMatch (문맥 맞춤 논문 찾아주기 서비스) - 기술 스택: React, Tailwind CSS (Lucide React 아이콘 사용 가능) [구현해야 할 화면 및 기능 요구사항] 1. 메인 검색 페이지 (Tailwind CSS로 깔끔하고 현대적인 UI) - 상단: 서비스 로고 및 타이틀 "PaperMatch"와 서비스 소개 문구 - 중앙: 사용자가 상황을 자유롭게 적을 수 있는 대형 텍스트 박스 (Placeholder: "현재 연구 중인 주제나 필요한 논문의 맥락을 자유롭게 입력하세요. 예: 학부 졸업 논문으로 대형 언어 모델(LLM)의 할루시네이션 현상을 해결하는 프롬프트 엔지니어링 기법에 대해 쓰고 있어. 최신 논문이 필요해.") - 필터 옵션: 발행 연도(최근 1년, 최근 3년, 최근 5년, 전체), 언어(영어, 한국어, 전체) 드롭다운 메뉴 - 검색 버튼: 클릭 시 입력 검증 진행 (글자 수가 너무 짧으면 "상황을 조금 더 구체적으로 적어주세요"라는 부드러운 Alert 또는 Toast 메시지 노출) 2. 로딩 화면 (시뮬레이션) - 검색 버튼 클릭 후 3초 동안 "AI가 상황을 분석하고 논문을 찾는 중입니다..." 문구와 함께 부드러운 스피너(Spinner) 애니메이션 노출 3. 결과 화면 (검색 및 추천 결과 리스트) - 사용자가 입력한 문장에서 추출한 '핵심 키워드'를 상단에 태그 형태로 표시 (예: #LLM, #Hallucination, #Prompt_Engineering) - 추천 논문 리스트(3개) 노출. 각 논문 아이템은 다음 정보를 포함: * 논문 제목, 저자, 발행일, 초록(Abstract) 요약 * [AI 추천 이유]: "이 논문은 사용자의 [입력 상황]에 맞추어 ~한 이유로 추천합니다." 형식의 상세한 요약 문구 * [원문 링크] 버튼 (구글 스칼라 등으로 연결되는 가짜 링크 또는 실제 구글 스칼라 검색 링크) * [북마크(저장)] 버튼 (클릭 시 하트/별 아이콘이 채워지며 저장됨) * [인용 복사] 버튼: 클릭 시 해당 논문의 APA 양식 출처가 클립보드에 복사되고 "복사되었습니다" 알림 표시 4. 북마크(보관함) 사이드바 또는 탭 - 사용자가 북마크한 논문들을 모아서 볼 수 있는 별도의 섹션 제공 사용자가 바로 서비스를 사용할 수 있는 수준으로 UI와 인터랙션을 완성도 높게 짜줘. 코드 작성을 시작해줘.
